export const HexStrip = ({ colors, showNumbers }) => (
  <div style={{ display: 'flex', gap: '2px' }}>
    {colors.map((hex, i, arr) => (
      <div key={i} style={{ flex: 1, display: 'flex', flexDirection: 'column', alignItems: 'center', gap: '2px' }}>
        <div style={{
          width: '100%', height: '2rem',
          background: hex,
          borderRadius: i === 0 ? '4px 0 0 4px' : i === arr.length - 1 ? '0 4px 4px 0' : '0',
        }} title={hex} />
        {showNumbers && (
          <span style={{ fontSize: '9px', color: 'var(--mono-text-low-contrast)', lineHeight: 1 }}>{i + 1}</span>
        )}
      </div>
    ))}
  </div>
);

# Colors

Built on [Radix Colors](https://www.radix-ui.com/colors) — a 12-step scale
designed for accessible UI. See the
[Radix scale anatomy](https://www.radix-ui.com/colors/docs/palette-composition/understanding-the-scale)
for the full rationale behind each step.

**Source:** `src/styles/theme/colors.scss`

## Using the scale

Each tone provides 12 tokens following the pattern `--{tone}-{step-name}`.
The 12 steps are grouped into functional roles
([Radix scale anatomy](https://www.radix-ui.com/colors/docs/palette-composition/understanding-the-scale)):

- **Steps 1–2 — Backgrounds.** App background and subtle background. Use for page fills and
  lightly tinted sections.
- **Steps 3–5 — Component backgrounds.** Base, hover, and active states for interactive
  elements like cards, inputs, and list items.
- **Steps 6–8 — Borders.** Subtle separators, default borders, and focused/hovered borders.
  Designed to be visible on steps 1–5 in both themes.
- **Steps 9–10 — Solid fills.** High-chroma fills for buttons, badges, and indicators.
  Step 10 is the hover state. These have WCAG AA contrast against white text
  (with light-mode overrides where needed).
- **Steps 11–12 — Text.** Low-contrast text (labels, secondary copy) and high-contrast
  text (headings, body). Step 11 meets 4.5:1 on steps 1–3.

### Examples

```css
/* Card with accent tint */
.card {
  background: var(--accent-bg);
  border: 1px solid var(--accent-border-subtle);
  color: var(--accent-text);
}

/* Solid button */
.button {
  background: var(--accent-solid);
  color: var(--accent-on-solid);
}
.button:hover {
  background: var(--accent-solid-hover);
}

/* Error state */
.input--error {
  border-color: var(--error-border);
  color: var(--error-text-low-contrast);
}
```

Each tone also defines `--{tone}-on-solid` for text on solid backgrounds
(white for most tones, dark for secondary and warning).

## Accessibility

Light mode includes WCAG AA contrast overrides for steps 9 and 11, ensuring
at least 4.5:1 contrast ratio against step-3 backgrounds and white foregrounds.
These are defined in the `:root:not(.dark)` block in `colors.scss`.
