# Energy Model

Core data types and conventions used across the dashboard.

## Settlement periods

All energy data uses **half-hourly settlement periods** — the standard interval
for UK electricity markets. Timestamps represent the **start** of a period and
cover `[ts, ts + 30 min)`.

```ts
const HALF_HOUR_MS = 1_800_000;
```

## Core types

### PricePoint

A single half-hourly price from the Octopus Agile tariff.

```ts
interface PricePoint {
  ts: number;    // Unix-ms, start of period
  price: number; // pence/kWh inc. VAT
}
```

Fetched from `/api/agile-prices` and normalised by `mapAgilePrices()`.

### HouseholdUsageRow

Per-slot consumption for three household profiles.

```ts
interface HouseholdUsageRow {
  ts: number;
  standard: number;        // kWh
  heatPump: number;        // kWh
  heatPumpBattery: number; // kWh
}
```

Parsed from `/data/household_usage.csv` by `parseHouseholdUsageCsv()`. CSV
time-only values (e.g. `"06:00"`) are anchored by `resolveUsageAnchor()` to a
day the price feed actually covers — today when fully published, otherwise the
most recent complete day — so every slot maps to a real price.

### HouseholdKey

```ts
const ALL_HOUSEHOLD_KEYS = ['standard', 'heatPump', 'heatPumpBattery'] as const;
type HouseholdKey = (typeof ALL_HOUSEHOLD_KEYS)[number];
```

### TimeRange

An inclusive time window: `[fromTs, toTs]`.

```ts
interface TimeRange {
  fromTs: number; // inclusive
  toTs: number;   // inclusive
}
```

Both `computeFullRange` (which sets `toTs` to the last data point) and chart
selections (which snap to the nearest data point) produce inclusive ranges.
Consumer functions use `lowerBound` for the start and `upperBound` for the
end to include elements at both boundaries.

> **Note:** `FlexEvent.startTs` / `endTs` use half-open `[startTs, endTs)`
> semantics — this is separate from `TimeRange`.

## Household themes

Each household maps to a display label, Radix colour tone, and CSS custom
property variables for stroke and fill. The tone flows through the entire UI
via BentoTile CSS custom property remapping.

```ts
const HOUSEHOLD_THEMES = {
  standard:        { label: 'Standard',            tone: 'accent',    strokeVar: 'var(--accent-solid)',    fillVar: 'var(--accent-solid)' },
  heatPump:        { label: 'Heat Pump',           tone: 'secondary', strokeVar: 'var(--secondary-solid)', fillVar: 'var(--secondary-solid)' },
  heatPumpBattery: { label: 'Heat Pump + Battery', tone: 'warning',   strokeVar: 'var(--warning-solid)',   fillVar: 'var(--warning-solid)' },
};
```

A companion `TONE_VARS` map covers all five `ChartTone` values (`accent`,
`secondary`, `positive`, `negative`, `warning`) and is used by chart components
for series stroke and tooltip swatch colours.

**Source:** `src/config/households.ts`

## Data loading

`useMarketData()` fetches prices, flex events, and usage in parallel, returning
a discriminated union: `loading` → `ready` or `error`. Components switch on
`status` to render skeletons, error states, or data views.

## API route

`/api/agile-prices` proxies the Octopus Energy Agile tariff. The current Agile
product is discovered from the Octopus product list (`selectAgileProductCode`),
falling back to a known code; the region is fixed (L). When the upstream is
unreachable it serves a bundled real-price snapshot (`source: 'snapshot'`,
flagged in the UI by `SampleDataBanner`) so the dashboard always renders. ISR
revalidates every 15 minutes.

**Source:** `src/app/api/agile-prices/route.ts`, `src/utils/octopus.ts`
