# Motion

All animations are configured in **`src/config/motion.ts`** and mirrored as CSS custom properties in **`src/styles/theme/motion.scss`**. This keeps timing, easing, and distances consistent and tuneable from a single source.

Animations respect `prefers-reduced-motion` automatically — Motion.js handles this for JS-driven animations, and CSS animations use a `@media` query fallback.

## Presets

### Reveal (viewport-triggered entrance)

Elements animate in when they scroll into view. Triggers once, never replays.

```tsx
import { reveal, stagger } from '@/config/motion';

<motion.div
  variants={reveal.fadeUp}
  initial="hidden"
  whileInView="visible"
  viewport={reveal.viewport}
  transition={reveal.transition(index * stagger.normal)}
/>
```

**Variants available:**

| Variant | Effect |
|---------|--------|
| `reveal.fadeUp` | Opacity 0 → 1, Y +20px → 0 |
| `reveal.fade` | Opacity only |
| `reveal.scaleUp` | Opacity + scale 0.95 → 1 |

### Swap (content enter/exit in place)

For replacing content within a container — results, state changes.

```tsx
import { swap } from '@/config/motion';

<AnimatePresence mode="wait">
  <motion.div
    key={uniqueKey}
    initial={swap.enter}
    animate={swap.visible}
    exit={swap.exit}
    transition={swap.transition}
  />
</AnimatePresence>
```

| Property | Value |
|----------|-------|
| Enter | opacity: 0, scale: 0.95, y: +6 |
| Visible | opacity: 1, scale: 1, y: 0 |
| Exit | opacity: 0, scale: 0.95, y: -6 |
| Duration | 0.2s (fast) |

**Used by:** `ShiftSimulator` result display

---

### Crossfade (skeleton to content)

Skeleton overlays the content as an absolute-positioned layer, then fades out when loading completes. Content is already laid out underneath — no layout shift.

```tsx
import { crossfade } from '@/config/motion';

<AnimatePresence initial={false}>
  {loading && (
    <motion.div
      key="skeleton"
      className={styles.skeletonOverlay}
      exit={crossfade.exit}
      transition={crossfade.transition}
    >
      {skeleton}
    </motion.div>
  )}
</AnimatePresence>
```

| Property | Value |
|----------|-------|
| Exit | opacity: 0 |
| Duration | 0.4s (normal) |

**Used by:** `BentoTile` loading state

### Draw line (SVG chart animation)

Chart series lines draw themselves left-to-right using `stroke-dashoffset`. Pure CSS animation — no Motion.js needed.

<LineDrawDemo />

```tsx
import { drawLine } from '@/config/motion';

// Applied via CSS custom properties on the SVG path:
style={{
  '--line-length': pathLength,
  '--line-delay': `${index * drawLine.stagger}s`,
  '--draw-line-duration': `${drawLine.duration}s`,
}}
```

The CSS animation references these variables:

```scss
.seriesLineAnimated {
  stroke-dasharray: var(--line-length);
  stroke-dashoffset: var(--line-length);
  animation: drawLine var(--draw-line-duration) var(--line-delay) var(--ease-out) forwards;
}
```

| Property | Value |
|----------|-------|
| Duration | 1s |
| Stagger | 0.15s between series |
| Easing | ease.out |

**Used by:** `TimeSeriesChart` series lines

## CSS custom properties

All motion tokens are available as CSS custom properties for use in SCSS or inline styles:

```scss
// In any SCSS file:
animation: myAnim var(--duration-normal) var(--ease-out);
transition: opacity var(--duration-fast) var(--ease-out);
```

## Adding a new animation

1. Define timing/easing values in `src/config/motion.ts`
2. If CSS-only, mirror the values as custom properties in `src/styles/theme/motion.scss`
3. Import from `@/config/motion` in your component
4. Use `prefers-reduced-motion` media query for CSS animations, Motion.js handles it automatically for JS
5. Update this page with the new preset
