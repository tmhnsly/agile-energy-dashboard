# Progressive Web App (PWA)

The dashboard is a fully installable PWA with offline support, giving users a
native-app experience on any device.

## What you get

- **Installability** ‚Äî "Add to Home Screen" on mobile and desktop via the web
  app manifest.
- **Offline support** ‚Äî cached pages and data remain accessible when the
  network drops.
- **Background caching** ‚Äî static data and API responses are cached with
  strategy-appropriate TTLs.
- **Theme sync** ‚Äî the browser chrome colour tracks the current light / dark
  theme.

---

## Architecture

PWA support is powered by [Serwist](https://serwist.pages.dev/) (a maintained
Workbox successor) via its `@serwist/next` integration.

1. **Build time** ‚Äî `@serwist/next` wraps the Next.js webpack config to compile
   `src/sw.ts` into `public/sw.js` and inject a precache manifest.
2. **Runtime** ‚Äî the service worker precaches app shell assets and applies
   runtime caching strategies for data and images.
3. **Dev mode** ‚Äî the service worker is **disabled** in development
   (`process.env.NODE_ENV === "development"`) so hot-reload works normally. Use
   `pnpm dev:pwa` (webpack dev server) to test SW behaviour locally.

---

## Caching strategies

| Resource | Strategy | Cache name | TTL / limit |
|---|---|---|---|
| App shell (JS, CSS, HTML) | **Precache** | `serwist-precache` | Versioned at build |
| `/data/*` (JSON, CSV) | **CacheFirst** | `static-data` | 30 days, max 32 entries |
| `/api/*` | **StaleWhileRevalidate** | `api-data` | 1 hour, max 32 entries |
| Fonts | **CacheFirst** | `fonts` | 1 year, max 16 entries |
| Images | **StaleWhileRevalidate** | `images` | 30 days, max 64 entries |
| Everything else | Serwist `defaultCache` | various | Serwist defaults |

## Manifest & icons

`src/app/manifest.ts` exports a typed Next.js metadata route that generates
`/manifest.webmanifest` at build time. Key fields:

- `id: "/"` ‚Äî stable PWA identity, independent of `start_url` changes.
- `display: "standalone"` ‚Äî hides the browser chrome when installed.
- `background_color` / `theme_color` ‚Äî set to the dark theme base (`#111113`).

## Offline experience

Three pieces work together:

1. **`src/app/~offline/page.tsx`** ‚Äî a full-page fallback served by the service
   worker when a navigation request fails and no cached page is available.
2. **`useNetworkStatus` hook** ‚Äî returns `true` / `false` based on
   `navigator.onLine`, updating reactively via `useSyncExternalStore`.
3. **`OfflineBanner` component** ‚Äî conditionally rendered in the dashboard shell
   when `useNetworkStatus()` returns `false`. Shows a top-of-page warning
   strip: *"You're offline ‚Äî showing cached data"*.

When the user goes offline mid-session, the banner appears and cached API data
continues to be served. If they navigate to a page that hasn't been cached, the
`~offline` fallback page is shown instead.

---

## Theme sync

`ThemeColorSync` is a render-nothing client component mounted inside
`Providers`. It reads the computed `--app-bg` CSS variable (Radix `slate-1`)
and updates `<meta name="theme-color">` so the browser chrome (address bar,
status bar) always matches the design-system background for the active theme.

On mount it replaces the server-rendered media-query meta tags with a single
authoritative tag controlled by `next-themes`' resolved theme, avoiding
conflicts when the user's system preference differs from their in-app choice.

## Build scripts

| Script | Runner | Service worker |
|---|---|---|
| `pnpm dev` | Turbopack | **Disabled** ‚Äî fastest DX |
| `pnpm dev:pwa` | Webpack | **Enabled** ‚Äî test SW locally |
| `pnpm build` | Webpack | **Enabled** ‚Äî production build |

`dev:pwa` exists because Serwist's Next.js plugin requires webpack (Turbopack
support is not yet available).
