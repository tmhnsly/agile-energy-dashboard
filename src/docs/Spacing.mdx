## Container query breakpoints

Tile-scoped, inline-size based. Requires the ancestor `BentoTile` to set
`container-type: inline-size` and `container-name: tile`.

```scss
@use 'mixins/container-queries' as cq;
```

<div style={{ margin: '0.5rem 0 1.5rem' }}>
  <CQBreakpoint label="cq.small" width={290} activates="Standard tile @ tablet" drives="Base text, compact spacing" />
  <CQBreakpoint label="cq.medium" width={340} activates="Standard tile @ desktop 1280px" drives="Stat values sm→lg, chart heights sm→md" />
  <CQBreakpoint label="cq.large" width={640} activates="Wide / feature tile" drives="Multi-column layouts" />
  <CQBreakpoint label="cq.xlarge" width={900} activates="Wide tile @ desktop" drives="Stat values at 2xl, max spacing" />
</div>

`cq.at($w)` is available as an escape hatch for one-off widths (e.g. `cq.at(475px)` for the StatsBar 3-card row).

**Tile content-box widths** for reference:
- **Standard (1col):** 294px @ tablet · 345px @ desktop · 398px @ 1440px
- **Wide (2-3col):** 654px @ tablet · 1166px @ desktop · 1326px @ 1440px
